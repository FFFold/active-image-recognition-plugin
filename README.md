# 主动识图插件

关闭 MaiBot 的自动被动识图功能，为 Bot 提供主动识图工具 `recognize_image`，让 LLM 自己决定何时查看图片。

## 功能

- **关闭被动识图**：拦截入站消息中的图片，阻止自动 VLM 描述生成，节省 token 消耗
- **主动识图工具**：为 LLM 暴露 `recognize_image` 工具，Bot 可主动调用查看图片内容
- **带问题识图**：LLM 可传入 `question` 参数让 VLM 带着具体问题回答，而非仅获取通用描述
- **转发消息支持**：自动处理转发合集消息中的图片，同样赋予索引号
- **缓存管理**：自动缓存近期图片，支持容量上限和过期清理

## 安装

将插件目录放入 MaiBot 的 `plugins/` 目录下，在 WebUI 插件管理页面中启用即可。

## 配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `plugin.enabled` | bool | false | 是否启用插件 |
| `cache.max_images` | int | 200 | 最大缓存图片数量 |

## 使用

启用插件后，Bot 收到的图片会被替换为 `[图片 #N]` 占位符（N 为递增编号），不再自动调用 VLM 描述。

当 LLM 需要查看某张图片时，可调用 `recognize_image` 工具，传入图片编号，工具会返回图片的详细描述。

### 识图提示词

插件默认使用 MaiBot 框架的 `prompts/{locale}/image_description.prompt` 作为 VLM 识别提示词。如果用户在 `data/custom_prompts/{locale}/image_description.prompt` 放置了自定义覆盖文件，插件会优先使用自定义版本。两者都不可用时回退到内置默认文本。

## 工具

### `recognize_image`

- **参数**：
  - `image_number`（整数，必填）— 要识别的图片编号
  - `question`（字符串，可选）— 对图片的具体问题，例如「这是什么品牌的产品？」。不提供则使用默认识图提示词
- **返回**：图片的 VLM 描述文本，附带图片数据用于多模态回显
- **触发方式**：LLM 主动调用

## 注意事项

1. 插件仅缓存启用后收到的图片，历史图片无法通过此工具识别
2. 图片数据存储在内存中，重启后清空
3. 同一图片多次发送会占用多个缓存槽位（不同编号）
4. 部分框架版本下 LLM 需先调用 `tool_search` 发现该工具后才能使用
